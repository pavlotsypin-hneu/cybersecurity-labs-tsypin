# Демонстрація SQL-інʼєкцій (Vulnerable vs Safe)

Навчальний консольний застосунок для демонстрації уразливостей SQL-інʼєкцій та методів захисту від них на прикладі авторизації та пошуку даних.

## Мета
Показати, як SQL-інʼєкції працюють у вразливих запитах з прямим підставленням користувацького вводу та як вони блокуються при використанні параметризованих запитів.

## Технології
- Python  
- SQLite  
- sqlite3  

## Опис роботи
Застосунок створює просту базу даних з персональними даними користувачів і студентів. Реалізовано дві версії функціоналу: вразливу та захищену. У вразливій версії введені користувачем дані напряму підставляються в SQL-запит, що дозволяє виконувати SQL-інʼєкції. У захищеній версії використовуються параметризовані запити, які запобігають зміні логіки SQL.

## Демонстрація атаки
- **Вразлива авторизація:** payload `admin'--` дозволяє увійти без знання пароля.  
- **Вразливий пошук:** payload `' OR 1=1--` повертає всі записи з таблиці студентів.  
- **Захищена версія:** ті самі payload-и не працюють і не призводять до витоку даних.
